# Q1 (RESOLVIDO) — Grafo de relações (T3.2 / audit 5.2): reactflow vs SVG inline

**Decisão (Marco, 2026-07-13): SVG inline aprovado.** Motivos reforçados:
1. reactflow são 2 deps na prática (canvas + layout separado) — custo escondido.
2. Sem verificação no browser, escolha o que dá pra inspecionar programaticamente (SVG DOM > portais/canvas).
3. 50 nós é small — SVG nativo aguenta 500+ sem esforço.
4. Melhor sinal de portfólio (viz autoral).
5. Bundle + CSS global do reactflow são fricção.

**Regra derivada pra Q# futuras:** sem acesso visual, prefira sempre a opção verificável sem browser.

## Complementos exigidos ao plano (parte da DoD da T3.2)
- **Layout circular com peso:** ordenar personagens por `last_seen_turn` desc; rotacionar labels pelo ângulo; raio levemente maior pra quem tem >5 relações.
- **Interação mínima:** hover no nó → destaca só as arestas dele (dim as outras); click no nó → scroll do chat pro 1º turno em que ele aparece (mesma feature do T3.1).
- **Legend obrigatório:** cor por `kind`, espessura por `|valence|`.
- **Empty state informativo** (não `<svg/>` vazio).
- Se a legibilidade travar em uso real: migrar pra force-directed **dentro do SVG** com `d3-force` como dep de *compute* (lê posições) — zero reactflow.

## Comparação original (para registro)

| Critério | reactflow (`@xyflow/react`) | SVG inline |
|---|---|---|
| Dependência nova | Sim (~40kb+ gzip + CSS própria) | Nenhuma |
| Layout automático | Não vem pronto — precisa `d3-force`/`elkjs`/`dagre` | Eu implemento (circular trivial; force trabalhoso) |
| Pan/zoom/drag/minimap | Pronto | Eu construo (ou não tem) |
| Legibilidade a 30–50 nós | Melhor (force espalha) | Circular emaranha no centro |
| Tema dark/light | Mapear CSS vars deles | Trivial (nossas vars) |
| Testável no Vitest | Opaco (portais/canvas) | Fácil (`<circle>`/`<line>`) |
| Risco sem browser | Maior | Menor |
| Licença | MIT (não é blocker) | N/A |
| Sinal de portfólio | "usei lib" | "viz autoral" |

**Recomendação (aceita):** SVG inline pro MVP; migrar pra force-directed-em-SVG só se a legibilidade travar.
